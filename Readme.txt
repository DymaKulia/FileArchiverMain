Для проверки проделанной работы по Web Start технологии необходимо запустить на локальном ПК Wep application

"ServerForFileArchiverMain" (для этого поместить файл ServerForFileArchiverMain.war в папку 

[контейнер веб приложений(папка томката например)]\webapps) и запустить сервер веб приложений.

Файл ServerForFileArchiverMain.war находится на GitHub.


После запуска сервера запустить браузер (IE-предпочтительно) и набрать в адресной строке 

http://localhost:8080/ServerForFileArchiver. После этого на экране отобразятся 2 ссылки на 2 приложения

FileArchiverMain 1.0.0 : download 
ConfigFileArchiverMain 1.0.0 : download 


FileArchiverMain 1.0.0 представляет из себя приложение, которой производит непосредственно архивацию

в соответствии с файлами настройки которые хранятся по адресу С:\FileArchiver


ConfigFileArchiverMain 1.0.0 представляет собой приложение, которое позволяет сформировать файлы 

настройки необходимые для работы приложения FileArchiverMain 1.0.0 и сохраняет их соответственно в папку

С:\FileArchiver


После загрузки приложений можно создать ярлыки на рабочем столе для них. Для этого нужно зайти в Java Controle Panel

(Win7 Control panel -> Programs -> Java) на вкладке General найти кнопку View и нажать. Откроется окно в котором можно

будет найти кнопку для создания ярлыков для скачанных приложений.


После того как ярлык для FileArchiverMain будет создан его можно будет запускать из консоли или .bat файла с содержанием

start [путь к ярлыку]\[имя ярлыка].lnk

Важно! Для корректной работы без необходимости подтверждения запуска приложения необходимо еще скачать сертификат Dyma.cer
и импортировать его. Для этого пройти Java Controle Panel->вкладка Security -> Manage Certificates ->Certificate type выбрать 
Signer CA и импортировать скачанный Dyma.cer.


Если Будет проверяться работа непосредственно архиватора и приложения позволяющего его сконфигурировать то достаточно с GitHub
скачать 2 архива Archiver-0.0.1-archiver и Archiver-0.0.1-archiverConfig и запустить.
